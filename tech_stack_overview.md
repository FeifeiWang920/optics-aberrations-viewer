# 《几何光学像差》教学网 - 技术栈与依赖说明

为了便于您后续对该项目进行维护和交接，以下是我们搭建这个网站所使用的详细技术架构以及包管理方式说明：

## 一、 核心技术栈 (Tech Stack)

本项目采用目前世界上最流行、最现代化的前端 Web 技术进行构建：

1. **核心框架**：**Next.js 15** (基于 React 19)
   * *在本项目中的作用*：它就像是一个脚手架，提供了路由（将不同像差切分为独立页面）、服务端渲染、以及极快的打包构建系统。
2. **编程语言**：**TypeScript**
   * *在本项目中的作用*：带类型系统的 JavaScript。它能让代码在运行前就被查出很多拼写或逻辑错误，是工业界大型项目的标配。
3. **样式库**：**Tailwind CSS (v4)**
   * *在本项目中的作用*：它让我们能直接在代码里写像 `glass`、`text-accent-cyan` 这样的类名来进行样式设计，且最终打包产物的体积非常小。
4. **动画库**：**Framer Motion**
   * *在本项目中的作用*：用于实现我们首页里的卡片滑入、平滑过渡，以及我们在光线追踪引擎中射线沿路径“飞出”的酷炫动效。
5. **图标库**：**Lucide React**
   * *在本项目中的作用*：提供了首页以及导航栏中使用的高清矢量小图标（SVG Icons）。

## 二、 运行项目需要哪些工具

要跑起这套系统，这主要分为**系统级别的工具**和**项目级别的库**。

### 1. 系统级工具 (全局依赖)
你需要在一台干净的电脑上安装 **Node.js**。这是整个前端生态的运行环境。
* 下载安装 Node.js 后，系统就会自带 `npm`（Node Package Manager），这也是我们用来下载所有第三方库的工具。
* **安装位置**：**安装在 Windows 系统目录中**（C盘或您指定的位置）。它是全局生效的，只要安装了一次，以后做任何前端开发都不需要再重新安装。

### 2. 项目级库 (局部依赖)
这就是上面提到的 Next.js, React, Framer Motion 等源代码模块。
* **获取方式**：在拥有项目的文件夹下终端运行 `npm install`。
* **安装位置**：**全部集中安装在当前项目目录下的 `node_modules` 文件夹里**（即 `e:\Code\几何光学像差\node_modules`），完全不会污染您的系统目录（System Directory）！

## 三、 本地安装逻辑解析

前端工程化的核心思想之一就是：**局部（本地）安装为主，全局系统为辅**。

1. **完全的自包含 (Self-contained)**：当前项目中使用的所有库版本的精确定义都写在了根目录的 `package.json` 文件中。不论是将项目转移到其他电脑，甚至发给了学生，只要对方装了 Node.js，执行一下 `npm install` 就能在他的电脑项目的 `node_modules` 中复刻一模一样的环境，保证了 100% 能跑起来。
2. **互不干扰**：因为依赖仅存在于该项目文件夹下，如果您再去写另一个网站，需要旧版本的 React，它们之间绝对不会互相冲突打架。
3. **Git 忽略**：由于 `node_modules` 里包含了成千上万个极小的文件模块，总体积可能有几百 MB，为了上传 GitHub 的速度，我们在创建项目时默认生成了 `.gitignore` 文件。它会指示 Git 工具**主动忽略掉整个包库文件夹**。因此，最终您推送到网络上的只有您和我写的纯文本代码（仅仅几百 KB）。

### 总结
你当前系统的干净程度被完美保持：**您的电脑上只在系统级别安装了一个 `Node.js`。所有构成这个酷炫光影界面的 Next.js、React 等几十上百个库，全都如同绿色软件一样被封装在您项目目录树里的 `node_modules` 下。**
